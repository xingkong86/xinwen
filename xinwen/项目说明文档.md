# 新闻爬取与数据可视化系统 - 项目说明文档

## 项目概述

**项目名称：** 新闻爬取与数据可视化系统

**项目核心功能概述：** 本系统是一个全栈新闻数据采集与分析平台，实现了从新浪新闻这个数据源自动爬取新闻数据、智能提取关键词、存储到数据库，并通过可视化大屏展示新闻统计趋势、热门关键词词云、热点事件时间线等功能。系统支持定时自动爬取和手动触发爬取，为新闻舆情监测和数据分析提供完整解决方案。

**主要技术栈：**
- **后端：** Python 3.x、FastAPI、SQLAlchemy 2.0（异步）、APScheduler、httpx、jieba
- **前端：** Vue 3、Vite、Element Plus、ECharts、Axios
- **数据库：** MySQL（通过 aiomysql 异步驱动）
- **开发工具：** python charm、webstorm

## 1. 系统架构概述

### 1.1 架构设计理念

本系统采用**前后端分离**的架构风格，后端提供 RESTful API 服务，前端通过 HTTP 请求与后端交互。系统设计遵循**高内聚、低耦合**原则，将数据采集、业务逻辑、数据访问、可视化展示等职责清晰分离。

核心设计理念：
- **异步优先：** 后端全面采用异步编程模型（async/await），提升并发处理能力
- **模块化设计：** 功能模块独立封装，便于维护和扩展
- **可扩展性：** 支持多新闻源接入，易于添加新的数据源
- **实时性：** 定时任务自动采集，前端实时刷新展示

### 2.2 模块功能详述

#### 模块一：新闻爬虫服务模块 (services/spider.py)

**核心职责：**
- 从新闻源 API（如新浪新闻）异步爬取新闻数据
- 使用 jieba 进行中文分词和关键词提取
- 数据去重（按标题判断是否已存在）
- 将处理后的新闻数据保存到数据库

**接口/API 简述：**

| 函数名 | 用途 |
|--------|------|
| `fetch_and_save_news(db, source)` | 核心入口函数，爬取指定来源的新闻并保存 |
| `fetch_sina_news(db, page, num)` | 爬取新浪新闻 JSON 接口 |
| `extract_keywords(text, top_n)` | 从文本中提取关键词（TextRank + TF-IDF） |
| `check_news_exists(db, title)` | 检查新闻是否已存在（去重） |
| `parse_timestamp(intime)` | 解析时间戳字符串 |

**依赖关系：**
- 依赖 `httpx` 进行 HTTP 请求
- 依赖 `jieba` 进行关键词提取
- 依赖 `crud.py` 的 `create_news()` 保存数据
- 依赖 `models.py` 的 `NewsItem` 模型

---

#### 模块二：API 路由模块 (routers/news.py)

**核心职责：**
- 定义 RESTful API 端点
- 处理前端请求，调用业务逻辑
- 返回 JSON 格式的响应数据
- 提供新闻数据的增删改查和统计分析接口

**接口/API 简述：**

## 4. 数据库设计

### 4.1 设计概述

**数据库选型：** MySQL 关系型数据库

**设计原则：**
- 采用关系型数据库存储结构化新闻数据
- 使用索引优化查询性能
- 支持异步访问（aiomysql 驱动）
- 字段设计考虑扩展性（如 source 字段支持多来源）

### 4.2 核心数据实体

本系统核心数据实体为 **新闻条目 (NewsItem)**，对应数据库表 `news_item`。

### 4.3 表结构说明

#### 表名：news_item

| 字段名 | 数据类型 | 说明 | 约束 |
|--------|----------|------|------|
| `id` | INTEGER | 新闻ID，主键 | PRIMARY KEY, AUTO_INCREMENT, INDEX |
| `title` | VARCHAR(255) | 新闻标题 | NOT NULL, INDEX |
| `intro` | TEXT | 新闻简介或原标题 | NULLABLE |
| `keywords` | VARCHAR(255) | 分词后的关键词（逗号分隔） | NULLABLE |
| `source` | VARCHAR(50) | 新闻来源（如：sina, netease, toutiao） | NULLABLE, INDEX |
| `publish_time` | DATETIME | 新闻发布时间 | NULLABLE |
| `crawl_time` | DATETIME(timezone=True) | 新闻爬取时间 | DEFAULT NOW() |

### 4.4 关键索引

| 索引名 | 索引字段 | 索引类型 | 用途说明 |
|--------|----------|----------|----------|
| PRIMARY | `id` | 主键索引 | 唯一标识新闻记录 |
| idx_title | `title` | 普通索引 | 加速标题查询和去重判断 |
| idx_source | `source` | 普通索引 | 加速按来源统计查询 |
| idx_publish_time | `publish_time` | 普通索引 | 加速按日期范围查询和排序 |
| idx_crawl_time | `crawl_time` | 普通索引 | 加速最新新闻查询 |

```
                          └─────────────────┘

┌─────────────────┐       ┌─────────────────┐
│   keyword       │       │ news_keyword    │
│   (关键词表)     │       │ (关联表)         │
├─────────────────┤       ├─────────────────┤
│ id (PK)         │◄──────│ keyword_id (FK) │
│ word            │  M:N  │ news_id (FK)    │
│ frequency       │       └─────────────────┘
└─────────────────┘             ▲
                                │
                          ┌─────┴─────┐
                          │ news_item │
                          └───────────┘
```

**关系说明：**
- `news_source` → `news_item`：一对多关系，一个来源对应多条新闻
- `keyword` ↔ `news_item`：多对多关系，通过 `news_keyword` 关联表实现

### 5.1 后端部署

```bash
# 1. 创建虚拟环境
python -m venv .venv
source .venv/bin/activate  # Linux/Mac
# .venv\Scripts\activate   # Windows

# 2. 安装依赖
pip install fastapi uvicorn sqlalchemy aiomysql apscheduler httpx jieba pydantic

# 3. 初始化数据库
python init_db.py

# 4. 启动服务
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

```bash
# 1. 安装依赖
cd xinwen_qianduan
npm install

# 2. 开发模式运行
npm run dev

# 3. 生产构建
npm run build
```
